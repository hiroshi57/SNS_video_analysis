# NEXT_TASKS — SNS動画分析ツール 今後のアップデート

最終更新: 2026-06-13

現状の実装ベース:
- 取込: YouTube/他SNS(yt-dlp)/ローカルファイル → Files API or インライン
- 分析: Stage1(要約・文字起こし・章立て)→ Stage2(映像深掘り)→ 競合差別化、すべて Gemini Flash
- 速度優先モード: 10分超は文字起こしを抜粋・シーンを代表化して高速化
- UI: Streamlit、ステートマシン方式で段階ごとに進捗%表示、ダークテーマ固定
- 蓄積: SQLite(data/analyses.db)に分析結果を保存し同ジャンル競合比較

優先度: 🔴高 / 🟡中 / 🟢低　　工数目安: S(数時間)/ M(1日)/ L(数日)

---

## ① 分析時間を早める

- 🔴 **S — Files APIアップロードの永続キャッシュ**: 取込済み動画の再アップロードを避ける。
  現状の `ingest._upload_cache` はセッション内のみ。ファイルハッシュ→uriをDBに保存し分析跨ぎで再利用。
- 🔴 **M — Stage1/Stage2の並列実行**: 現状は直列。要約と映像深掘りは別プロンプトなので
  `concurrent.futures` で並行リクエストし体感時間をほぼ半分に(Flashのレート制限に注意)。
- 🟡 **S — 速度プリセットUI化**: `FAST_MODE_THRESHOLD_SEC` を画面から
  「高速/標準/高精度」3プリセットで選べるようにし、尺ごとに最適化できるようにする。
- 🟡 **M — 音声先行・映像後追い**: 長尺は先に音声抽出(ffmpeg)で文字起こしを即返し、
  映像分析を後追いでバックグラウンド実行。体感の初動を最速化。
- 🟢 **S — 結果キャッシュ**: 同一URL/ファイルハッシュの再分析はDBヒットで即表示しAPI呼び出しをスキップ。

## ② 分析精度を高める

- 🔴 **M — Pro/Flashの使い分け復活**: 課金有効化時に Stage2・競合比較だけ
  `gemini-2.5-pro` に上げる(.env の MODEL_DEEP 切替済み)。精度差を測るA/B評価セットを用意。
- 🔴 **M — structured output(response_schema)導入**: 現状はプロンプト指示+検証リトライ。
  google-genai の `response_schema` でスキーマを型強制し、欠落・破損を構造的に防ぐ。
- 🟡 **M — 視聴データの公式API連携(任意)**: 再生数・維持率は動画ファイルからは取れない。
  YouTube Data API等をオプション連携し「映像分析×実数値」で示唆を強化。
- 🟡 **S — 話者分離・専門用語辞書**: 話者ラベル精度向上+ジャンル別用語集をプロンプト注入し固有名詞誤りを削減。
- 🟢 **M — 評価ハーネス**: 代表動画10本の正解サマリで出力品質を自動採点し、プロンプト改善効果を定量化。

## ③ UI/UXを高める

- 🔴 **S — 結果画面に処理モード表示**: 高品質/速度優先のどちらで分析したかを結果タブにもバッジ表示(現状は進捗中のみ)。
- 🔴 **M — タイムライン連動**: 文字起こし・章・シーンの時刻クリックで動画が該当位置にジャンプ(st.video の start_time)。
- 🟡 **S — エクスポート**: 結果をMarkdown/PDF/Notionに書き出すボタン。Arc/やObsidian連携を想定。
- 🟡 **S — 比較ビュー**: サイドバー履歴から2本を選び横並び比較する専用画面。
- 🟢 **S — 部分再実行UI**: 各タブに「このタブだけ再分析」ボタンを置き、プロンプト微調整後の再生成を局所化。

## ④ 動画の作り変え示唆(リライト・改善提案)

> 「分析して終わり」でなく「次にどう作り直すか」まで踏み込む。本ツールの差別化の核。

- 🔴 **M — リライト提案タブの新設**: Stage2の強み/改善点を発展させ、
  「冒頭3秒のフック案を3パターン」「離脱しやすい区間と差し替え案」「テロップ追加箇所」
  「結論先出し版の構成案」を具体生成する新プロンプト(prompts/stage3_rewrite.md)。
- 🔴 **M — 競合の勝ちパターン抽出**: 同ジャンル蓄積から「伸びている型」を要約し、
  今回の動画に当てはめた改修指示を出す(compare.py の比較結果を入力に使う)。
- 🟡 **M — 台本・絵コンテ生成**: 改善案をもとに作り直し版の台本/シーン割りを下書き生成。
  ショート版への再編集案(尺圧縮・切り抜きポイント)も含める。
- 🟡 **S — サムネ/タイトル案**: 内容と訴求点からサムネ構図案・タイトル候補を複数提示。
- 🟢 **L — 画像生成連携**: サムネ案を画像生成APIでラフ出力(要API選定)。
