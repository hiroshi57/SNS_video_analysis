# Design.md — SNS動画分析ツール デザインシステム

> 目的: 分析結果を「ただ並べる」のではなく、**この動画の訴求ポイント・強みが一目で伝わる**
> プロフェッショナルな見せ方に統一する。UI 実装([app.py](app.py) の `inject_css` と
> `render_*`)はこの定義に従う。

---

## 1. デザイン原則

1. **メリハリ(視覚的階層)**: 重要なものは大きく・太く・色で立たせ、参照情報(文字起こし等)は
   トーンを落として静かに見せる。すべてを同じ強さで出さない。
2. **結論ファースト**: 「何を訴える動画か」「強みはどこか」を最上部の要点バンドで先に見せる。
3. **意味と色を対応させる**: 強み=緑、改善/弱点=琥珀、差別化/独自性=紫、参照情報=グレー。
4. **余白で区切る**: カードと余白でグルーピングし、罫線の多用を避ける。
5. **一貫性**: 同じ意味の要素は常に同じ部品(クラス)で表現する。

## 4. コア部品

- **要点バンド `.keypoints`**: 結果上部に3枚並べる。①🎯 訴求(何を伝える動画か) ②✨ 一番の強み
  ③📝 ひとことで(要約1行)。各カードは上にカラーのアクセントライン。
- **要約ヒーロー `.summary-hero`**: 3行要約を番号付きで大きめに。最重要行を最初に。
- **強みカード `.good`** / **改善カード `.warn`**: アイコン + 1項目1カード。色で意味を区別。
- **特徴カード `.vcard`**: 2列グリッド。ラベル(紫・太)+本文。
- **比較カード `.cmp`**: 軸ごとに1ブロック。「この動画 / 競合」を対比し、**差別化ポイントを強調枠**で締める。
- **文字起こし `.ts`(静的トーン)**: 時刻は等幅・弱色、本文は標準色。装飾は最小限にして
  「読む参照資料」として静かに見せる(メリハリの“弱”側)。

## 5. やらないこと

- 全項目を同じカード・同じ太さで均一に並べる(メリハリが消える)。
- 文字起こしを派手に装飾する(主役ではない)。
- 1画面に強い色を3種類以上ばらまく(視線が散る)。
